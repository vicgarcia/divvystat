divvystat.us : usage and rebalancing metrics

The line chart at the top of each station popup shows the number of bikes available at a station over the previous 72 hours. Data points are omitted when the count doesn't change, creating an effect that draws attention to periods of higher use.

The bar graph at the bottom of each popup shows the average usage per day of the week in the last 30 days. Usage is measured as the number of bikes removed from a station throughout the day. This metric is an approximate derrived from data collected from Divvy's website.

The chart and graph off the map show data on rebalancing efforts and "outages", which occur when a station is either out of service, has no bikes available, or has no dock spaces available for returns. The line chart shows the number of station outages over the last 72 hours. The bar graph shows the average number of concurrent outages by day of the week over the last 30 days.

This interactive application was originally created for the Divvy Data Challenge by Vic Garcia.

The source code is on available on GitHub.
